# エネルギー価格相関ダッシュボード 要件定義書

**バージョン:** 2.0
**最終更新:** 2026-03-14

---

## 1. プロジェクト概要

世界情勢（原油・LNG・石炭・為替・海運）が日本の消費者エネルギー価格（ガソリン・電気料金・ガス料金）に与える影響を可視化し、将来コストの予測を支援するWebアプリケーションを構築する。

燃料費調整制度のラグ構造（LNG高騰から電気代反映まで2〜5ヶ月）を明示的に扱い、「今の国際価格が将来の家計にどう影響するか」をひと目で把握できることを最重要目標とする。

---

## 2. 目的

- 複雑に絡み合うエネルギー指標の相関性と**因果の連鎖**を一目で把握できるようにする。
- 原料価格から国内価格への**タイムラグ（反映の遅れ）**を指標ごとに正確に可視化する。
- 「現在のLNG価格が続くと3ヶ月後の電気代・ガス料金はいくらか」という**将来コストのシミュレーション**を可能にする。

---

## 3. ターゲット・ユースケース

- **想定ユーザー:** エネルギー動向に関心の高い一般ユーザー。
- **活用シーン:**
  - 来週のガソリン価格動向の予測。
  - 数ヶ月後の電気料金、ガス料金動向の予測。

### 4.2 ラグ構造の定義

本システムが扱う主要なラグ（価格転嫁の遅れ）を以下のとおり定義する。

| 上流指標 | 下流価格 | ラグ期間 | メカニズム |
|---------|---------|---------|-----------|
| WTI原油 | ガソリン小売価格 | **1〜4週間** | 海上輸送・精製・卸売転嫁の遅延 |
| JKM LNG | 電気料金 | **2〜5ヶ月** | 燃料費調整制度（3ヶ月平均化＋2ヶ月遅延） |
| 豪州石炭 | 電気料金 | **2〜4ヶ月** | 同上（石炭は発電量の約31%を占める） |
| USD/JPY | ガソリン小売価格 | **数日〜1週間** | 元売り価格への即時転嫁 |
| USD/JPY | 電気料金 | **3〜5ヶ月** | LNG/石炭輸入コスト経由の間接影響 |
| JKM LNG | 都市ガス料金 | **2〜4ヶ月** | 原料費調整制度（2〜3ヶ月平均化＋1〜2ヶ月遅延） |
| USD/JPY | 都市ガス料金 | **2〜4ヶ月** | LNG輸入コスト経由の間接影響 |

## 5. 機能要件

### 5.1 メインチャート

- 収録全指標の時系列データを重ね合わせたマルチ軸チャート（過去最大10年分）。
- 各指標の表示ON/OFFを凡例クリックで切り替え可能。
- **正規化表示:** 特定の基準日を100とした指数表示への切り替え（指標間の変動率比較用）。
- **ラグシフト機能:** 指標ごとに独立したスライダーで表示をずらす。
  - ガソリン系：1〜8週間（週単位）
  - 電気代・ガス料金系：1〜6ヶ月（月単位）
  - 「相関が最も高くなるポイントを自動検出」ボタンを実装。

### 5.2 因果チェーンパネル

各指標間の因果関係とラグをフロー図で表示する。

```
Dubai原油高 ──[1〜2週]──▶ ガソリン値上がり
JKM LNG高  ──[3ヶ月平均化]──▶ 燃料費調整単価↑  ──[2ヶ月後]──▶ 電気代上昇
           ──[2〜3ヶ月平均化]──▶ 原料費調整単価↑ ──[1〜2ヶ月後]──▶ ガス料金上昇
円安進行   ──[数日]──▶ ガソリン値上がり
           ──[3〜5ヶ月後]──▶ 電気代上昇（遅延）
           ──[2〜4ヶ月後]──▶ ガス料金上昇（遅延）
```

- 現在の相関係数を矢印上にリアルタイム表示。
- クリックで当該指標ペアの詳細チャートに遷移。

### 5.3 相関行列（ヒートマップ）

- 収録全指標間の相関係数を色付けして表示（赤：正相関 / 青：負相関）。
- **ラグ付き相関モード:** 「3ヶ月前のLNG vs 現在の電気代」のように時間差を指定した相関も計算・表示。
- 期間フィルタと連動（選択期間内の相関を再計算）。

### 5.4 サマリーパネル

- 各指標の直近値・前日比・前週比・前月比を一覧表示。
- 「現在の上流価格が続いた場合の将来電気代・ガス料金・ガソリン価格の推定値」を並記（シミュレーション値と明記）。

### 5.5 予測シミュレーター

電気代・ガス料金それぞれの調整制度ロジックをUIに組み込み、以下を計算・表示する。

- 「現在のJKM LNG価格が今後も維持された場合、◯ヶ月後の電気料金は約◯円/kWh（±◯円）」
- 「現在のJKM LNG価格が今後も維持された場合、◯ヶ月後のガス料金は約◯円/m³（±◯円）」
  - 都市ガスは**原料費調整制度**（電気の燃料費調整制度に相当）で転嫁される。ラグは電気より短い（2〜4ヶ月）。
- 「今週のドル円レートが続いた場合、来週のガソリンは約◯円/L」
- ユーザーが独自の価格シナリオを入力して試算できる入力フォーム。

### 5.6 期間・表示制御

- **期間選択:** 1ヶ月、6ヶ月、1年、5年、10年、全期間、カスタム範囲。
- **発電ミックスオーバーレイ:** LNG/石炭/再エネ比率の帯グラフをメインチャートに重ねて表示（LNG相関の強度変化を把握するため）。

### 5.7 データ収集（バックエンド）

| データソース | 用途 | 認証 |
|------------|------|------|
| FRED API | 原油・為替・金利・石炭 | APIキー（無料） |
| EIA OpenData | LNG・石炭・国際価格 | APIキー（無料） |
| JEPX 公開データ | 卸売電力価格・発電ミックス | 不要（CSV公開） |
| e-Stat API | 消費者向け電気・ガス料金 | APIキー（無料） |
| GlobalPetrolPrices API | ガソリン・灯油小売価格 | APIキー（有料） |

- **バッチ処理:** 1日1回、深夜に全ソースから最新データを取得してDB更新。
- **差分更新:** 既存レコードとの差分のみ書き込み（APIリクエスト節約）。
- **エラーハンドリング:** ソース側の障害時は前回データを保持し、データ欠損を画面に明示。

## 6. UI/UX デザイン要件

- **レスポンシブ対応:** PC・スマートフォンの両方で最適表示。
- **インタラクティブ性:** チャート上ホバーで全指標の詳細値を縦断ツールチップ表示。
- **テーマ:** ダークモード／ライトモードの切り替え。
- **データ出典表示:** 各指標のソース名・最終更新日時を常時表示。
- **凡例:** ラグシフト適用中の指標は「（3ヶ月ラグ）」と明示。

## 7. 技術スタック

| レイヤー | 採用技術 | 理由 |
|---------|---------|------|
| Frontend | Next.js (App Router) | SSR・SEO・パフォーマンス |
| チャートライブラリ | Recharts | マルチ軸・カスタム軸に柔軟対応 |
| Backend API | Python / FastAPI | データ処理・相関計算に最適 |
| データ取得 | pandas + fredapi + eia-python | 各データソースの公式ライブラリ |
| 相関計算 | scipy.stats | バックエンドで事前計算しキャッシュ |
| Database | PostgreSQL + TimescaleDB拡張 | 時系列データの高速クエリ |
| スケジューラ | APScheduler / GitHub Actions | 日次バッチ処理（無料枠で対応可） |
| Deploy (Front) | Vercel | Next.jsとの親和性 |
| Deploy (API/DB) | Railway または Fly.io | PostgreSQL込みで低コスト |

## 8. 非機能要件

- **パフォーマンス:** 10年分の時系列データ（全指標）を3秒以内に描画すること。
- **正確性:** データのソース・最終更新日時を画面上に明記すること。ラグシフト適用中は必ずその旨を表示すること。
- **可用性:** データソース障害時も画面が壊れないこと（前回データで継続表示）。
- **セキュリティ:** APIキーは環境変数で管理し、フロントエンドに露出させないこと。

## 9. 実装フェーズ

| Phase | 内容 | 主な成果物 |
|-------|------|-----------|
| **Phase 1** | データ収集基盤の構築 | FRED・EIA・JEPXからのデータ取得スクリプト、DB設計・構築 |
| **Phase 2** | 基本チャート＋サマリーパネル | メインチャート（原油・LNG・石炭・為替・ガソリン・電気代・ガス料金）、指標ON/OFF、期間選択 |
| **Phase 3** | ラグ解析・相関行列 | ラグスライダー、自動最適ラグ検出、ラグ付き相関ヒートマップ |
| **Phase 4** | 因果チェーンパネル＋予測シミュレーター | フロー図、燃料費調整制度ロジック組込み、将来コスト試算フォーム |
